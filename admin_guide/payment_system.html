Payment System Administration
Payment Management
Viewing Transactions

Access all platform transactions from the Payment Management dashboard. Filter by status, date range, client, or lawyer.

Processing Payments

Monitor pending payments and ensure successful processing. The system automatically attempts to process payments three times before marking as failed.

Handling Refunds

Process refund requests through the transaction detail page. Partial or full refunds can be issued with appropriate documentation.

Financial Reporting

Generate financial reports for accounting purposes. Export transaction data in CSV or PDF format for further analysis.

Refund Process
1
Navigate to the transaction details page for the payment requiring a refund
2
Click the "Process Refund" button and select full or partial refund
3
Enter refund amount (for partial refunds) and document the reason
4
Review details and confirm the refund transaction
5
System processes the refund and notifies both client and lawyer

Payment Settings
Platform Fee Configuration

Adjust the platform's commission percentage from lawyer payments. Current default: 15% of transaction value.

Payment Gateway Settings

Configure payment processor settings, API keys, and webhook endpoints. The platform currently supports Stripe and PayPal integration.

Payout Schedule

Set the frequency of lawyer payouts (weekly, bi-weekly, or monthly) and minimum payout thresholds.

Fraud Detection

Configure fraud detection rules and thresholds. Review flagged transactions in the Fraud Monitoring section.
